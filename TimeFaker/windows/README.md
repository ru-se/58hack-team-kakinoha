# **Engineer Time - Windows Client Tools**

Windows PCの時刻を外部（WebSocketサーバー）からの指令に基づいて動的に変更するためのクライアントツールです。

## **1. アーキテクチャ構成**

本システムは、リアルタイム通信を担当するメインプログラムと、Windows OSの低レイヤー操作を行うユーティリティの2層構造で構成されています。

### **`time_client.py` (メインプログラム)**
サーバーからの指令を待ち受け、全体の制御ループを担当します。
- **リアルタイム通信**: `websockets` を使用し、サーバーと常時接続。
- **時刻同期ロジック**: 
    - 二重加算（ズレの累積）を防ぐため、常に **NTPサーバー (`ntp.nict.jp`)** から正確な基準時刻を取得。
    - 指令された `offset_minutes` をNTP時刻に加算して目標時刻を算出。
- **モジュール分割**: Windows APIの複雑な操作を隔離することで、メインロジックの可読性と保守性を高めています。

### **`windows_utils.py` (Windows API ユーティリティ)**
OS深部の操作（特権昇格、タイムゾーン変更）を抽象化したラッパー群です。

1. **特権の有効化 (`enable_privilege`)**
    - **役割**: プロセスに対し `SeTimeZonePrivilege`（タイムゾーン変更特権）を明示的に有効化します。
    - **必要性**: Windowsでは管理者権限であっても、事故防止のため時刻操作等の危険なスイッチはデフォルトでOFFになっています。これをONにしないとOSに操作を拒否されます。
2. **C言語構造体の定義 (`DYNAMIC_TIME_ZONE_INFORMATION` 等)**
    - **役割**: Pythonのデータを、Windowsカーネル（C言語）が理解できるメモリ形式に変換します。
    - **必要性**: Windows APIは厳格なデータレイアウトを要求するため、`ctypes` を使用して「1バイトの狂いもない」型紙を作成し、メモリレベルで正確に値を渡します。
3. **タイムゾーンの直接操作 (`SetDynamicTimeZoneInformation`)**
    - **役割**: 独自のタイムゾーン定義（Engineer Time）をOSに流し込み、表示時刻を強制変更します。
    - **必要性**: 標準の `tzutil` 等では既存の地域（ハワイ等）しか選べませんが、本APIにより「自由な分単位のオフセット」が可能になります。

## **2. 技術的なポイントと工夫**

### **① システム時刻ではなく「タイムゾーン」を操作**
当初検討した `SetSystemTime`（システムクロックの書き換え）は廃止し、**日本標準時（JST）の定義そのものを一時的に書き換える**手法を採用しました。
- **メリット**: UTC（絶対時刻）は正しいまま保持されるため、**Gitのコミット日時、ログの順序、HTTPS/OAuth認証（AWS/GCP等）への悪影響が一切ありません。**

### **② PyWin32 と CTypes のハイブリッド実装**
- **特権昇格**: 安定性の観点から `pywin32 (win32security)` を使用し、確実に権限を取得。
- **詳細制御**: タイムゾーンの微細な制御（夏時間自動調整の無効化など）が必要な部分は、C言語互換の `ctypes` を使用。
- この「いいとこ取り」により、堅牢な権限管理と柔軟な機能実現を両立させました。

### **③ 即時反映のブロードキャスト**
変更後、`SendMessage` を用いてシステム全体に `WM_SETTINGCHANGE` を通知しています。これにより、タスクバーの時計表示などを即座に更新させています。

## **3. セットアップと実行**

### **依存ライブラリのインストール**
```bash
pip install -r requirements.txt
```

### **実行環境**
- **OS**: Windows 10/11
- **管理者権限**: 時刻設定を書き換えるため、**「管理者として実行」したプロンプト**での実行が必須です。

### **実行方法**
```bash
python time_client.py
```

---

## **4. 通信プロトコル (JSON)**

クライアントは以下の形式のメッセージを受信して動作します。

```json
{
  "type": "OFFSET_UPDATE",
  "offset_minutes": 60,
  "reason": "DAILY_UPDATE"
}
```

- `offset_minutes`: 基準時間（NTP）から何分進めるか（正の値）または戻すか（負の値）。
