# LexiFlow Frontend — 現状まとめ

> 最終更新: 2026-02-20

---

## 概要

**LexiFlow** は、音声をリアルタイムで文字起こしし、IT 専門用語を自動検出・解説する Web アプリケーションです。

## 主要機能

### 1. 音声認識（`useSpeechRecognition`）

- Web Speech API（`webkitSpeechRecognition`）を使用
- 日本語（`ja-JP`）でリアルタイム文字起こし
- `continuous: true` / `interimResults: true` で連続認識
- `onend` で自動再起動（ブラウザの強制停止に対応）

### 2. 用語検出（`termDetection.ts`）

- `extractTerms`: テキストから IT 用語を抽出（英語・かな両対応）
- `highlightTerms`: テキスト内の用語をマークアップ（長い単語優先マッチ）

### 3. デモストリーミング（`useDemoStream.ts`）

- デモ用の仮想テキスト（`demo.ts`）を一定間隔でリアルタイム風に文字起こし出力
- 各チャンクの表示間隔を動的に調整可能
- スライダーや再生・停止ボタンなど、UIによる操作・進捗表示と連動

### 4. 用語データ（`terms.ts`）

- カテゴリ: `Frontend` / `Backend` / `Infra` / `AI/Data` / `General`
- 各用語に `shortDesc`・`longDesc`・`relatedTerms`・`externalUrl` を保持

### 4. バブルマップ（`BubbleCloud.tsx` / `TermBubble.tsx`）

#### バブルライフタイム管理（スマート削除 v3.0）

バブルの増えすぎによる視認性低下・パフォーマンス低下を防ぐため、厳密な個数・時間ベースの寿命管理を行っています。

| 条件                  | 挙動                                                                                                        |
| --------------------- | ----------------------------------------------------------------------------------------------------------- |
| バブル数 ≤ 20         | 削除なし。そのまま画面に残り続ける。                                                                        |
| バブル数 21〜30       | 最も古いバブルから順に「寿命カウントダウン」が開始。未ピン留めは5秒、ピン留め済みは10秒の寿命が設定される。 |
| バブル数 > 30（超過） | 即座に最も古いバブルから順に強制削除され、常に最大30個に保たれる。                                          |

**ゾンビバブル防止**
過去に一度でも抽出・生成された用語（バブル）は、テキストエリアがリセットされるまで内部でIDが記憶されるため、寿命により削除されたバブルが文字起こし文章から再抽出されて無限復活するのを防ぎます。

#### ピン留め機能（★）

- バブル右上の ☆ ボタン or 詳細パネルの「ピン」ボタンでトグル
- ピン中は黄色 ★（グロー付き）で強調表示
- ピン解除時はその時点から猶予タイマーが再スタート

#### バブル配置と重なり防止アルゴリズム（物理エンジン）

バブルの配置は単なるランダムや制的計算ではなく、コンポーネント内に実装された**2D物理シミュレーション**によって制御されています。

- **働く力（フォース）**:
  1. **浮力**: 下から上へ自然に浮かび上がらせる一定の力
  2. **中央への引力**: 画面外・端に張り付かないようにする求心力
  3. **反発力（ペナルティ法）**: バブル同士の距離が「お互いの半径の和＋マージン」未満になると、食い込んでいる深さに比例した力で互いを逆方向に押し出します。
- **動的リサイズへの対応**: 重要度（クリック数）の増加やピン留め状態のON/OFFによってバブルのサイズ（想定半径）が変更された瞬間、次のフレームの衝突判定により隣のバブルを物理的に押し退けて自動整列します。
- **高速描画**: `requestAnimationFrame` を用いて、Reactの再レンダリングを介さずDOMノードの `style.transform` を直接更新することで60fpsの動作を維持しています。

#### その他

- **自動切換えボタン**: 用語を自動ローテーション表示（1〜10 秒スライダー）

### 5. 動的レイアウトエンジン（`layout/`）

- VSCode / Unity 風のパネル自由配置
- ドラッグ＆ドロップでパネルを上下左右に移動・分割
- 分割線ドラッグでリサイズ
- 各パネルヘッダーの「×」ボタンから不要なウィンドウを削除可能
- 5 種類のプリセットレイアウト（デフォルト / 左右 / 2×2 / 横4列 / 縦4列）
- 全パネルの枠・ヘッダー・仕切り線がアクセントカラーと連動

### 6. その他 UI

- **ダークモード対応**（デフォルト: ダーク）
- **テーマカラー切替**: blue / indigo / purple / rose / emerald / orange
- **検索履歴パネル**: 最大 50 件、履歴内検索機能付き
