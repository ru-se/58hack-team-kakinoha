## 2. プロダクト概要

### どんな「本当に欲しい」に着目したか

- **誰**: 専門性の高い会話に参加する人（勉強会・学会・打ち合わせなど）
- **本当に欲しいこと**:
  - 音声が聞き取りづらくても内容を追いたい
  - 専門用語の意味を、会話の流れを止めずにすぐ確認したい
  - どの語が重要か分からず流れを見失いたくない
- **着目した点**: 「聞き取り」「難語の意味」「重要語の把握」という3つの詰まりどころを、**1本の導線**でまとめて解消する

### どんなアプローチで「本当に欲しい」を実現したか

- 音声を**リアルタイム文字起こし**し、テキスト上で**重要語・専門語を抽出・強調**
- 強調された語を**ワンタップで詳細表示**し、意味確認まで最短で到達
- **バブルマップ**で重要度を視覚化し、**ピン留め・履歴・辞書**で「後から振り返り」も可能に
- ログイン不要で、ブラウザだけで試せる

## 3. プロダクトの機能

（できるだけ具体的に列挙する）

- **音声認識・文字起こし**
  - Web Speech API（`ja-JP`）でリアルタイム文字起こし
  - デモ用のストリーミングテキスト入力
- **用語抽出・強調**
  - 文字起こしテキストから用語を検出
  - 重要度スコア（主題・会話との類似度、出現回数など）でランク付け
  - 文字起こし上の用語をハイライト表示（クリックで詳細）
- **用語マップ（バブルUI）**
  - 抽出用語をバブルで可視化、サイズで重要度を表現
  - カテゴリフィルタ（ALL / ピン中 / Frontend・Backend など）
  - バブル倍率スライダーで全体のサイズ調整
  - レイアウト変更時も枠に合わせてバブルがスケール
- **ピン留め・ピン中一覧**
  - 用語を右クリックなどでピン留め
  - IndexedDB に保存し、「ピン中」タブで表形式一覧（用語・カテゴリ・説明）
- **用語詳細・履歴**
  - 用語タップで詳細パネル（説明・カテゴリ・関連用語など）
  - 検索履歴（localStorage）で過去に確認した用語へ再アクセス
- **辞書管理（単語管理モーダル）**
  - 用語の一覧・編集・削除・一括登録
  - Backend 辞書 API 連携（あれば）
- **主題（キーワード）入力**
  - ハイライトしたいキーワードを入力し、主題ベクトルで用語の類似度・バブルサイズに反映
- **レイアウト**
  - 複数プリセット（デフォルト・左右分割・2×2・横4列・縦4列）で画面レイアウトを切り替え
- **その他**
  - ダークモード、設定モーダル、API 疎通確認用 UI など

## 4. 使い方・デモ

- **想定シーン**: 勉強会・セミナー・打ち合わせなど、専門用語の多い会話を聞きながらメモや理解を深めたいとき
- **基本操作**:
  1. 録音開始で文字起こしを開始
  2. 強調された用語をタップして詳細を確認
  3. 重要語はピン留めし、「ピン中」で一覧管理
  4. バブルマップで重要度を俯瞰し、倍率スライダーで見やすく調整
- **実行方法**（簡潔に）:
  - ローカル: Frontend `bun run dev` / Backend `uv run uvicorn main:app --reload` → `http://localhost:5173`
  - Docker: ルートで `make up` → Frontend `http://localhost:5173`、API `http://localhost:8000`
- **デモ動画**: 既存リンク（例: YouTube）をそのまま記載
- **画像・GIF**: 画面キャプチャ（文字起こし＋ハイライト、バブルマップ、ピン中表、詳細パネルなど）があるとよい

## 5. 技術面でのこだわり

- **フロント**
  - React 19 + TypeScript + Vite 6 + Tailwind v4 + Bun
  - Web Speech API でログイン不要・クライアント完結の文字起こし
  - バブル物理シミュレーション（衝突・浮力・中央寄せ）、ResizeObserver で枠サイズに応じたスケール、レイアウト変更時も基準幅をモジュール変数で保持して一貫した縮尺
- **バックエンド**
  - FastAPI + uv、形態素解析・ベクトル化 API（`/analysis/vectorize` 等）、主題ベクトル API、辞書管理 API（実装済みなら）
- **データ・永続化**
  - 履歴: localStorage / ピン留め: IndexedDB（idb）、辞書: Backend DB（スキーマ・CRUD があれば）
- **設計**
  - Frontend / Backend 責務分離、NLP を API で差し替え可能にし段階的に精度向上できる構成
- **難しかった点・チャレンジ**
  - バブルと枠のリサイズ・レイアウト切り替え時の一貫したスケール、ピン中とバブル表示の切り替え時の状態管理、IndexedDB と UI の同期 など（実際に苦労した点を簡潔に）

## 6. UI/UX 面でのこだわり

- 会話中でも**視線移動を抑えられるバブル中心 UI**
- **重要語の強調**と**詳細表示**を分離し、情報過多を避ける
- 文字起こしの用語ハイライトと「ピン中」表で**同じチップ風スタイル**を統一
- レイアウト・バブル倍率で**ユーザが自分で見やすさを調整**できる
- ダークモード、レイアウトプリセットで**利用シーンに合わせた見た目**
- 単語タップ→詳細→履歴・ピン留めまで**意味確認の導線を最短**に
