# 診断結果画面（Loading + 行動解析REPORT）

**パス:** `/result`

**前画面:** `/games/group-chat`（Game 3） → **現在:** `/result`

---

## フェーズ 1: Loading画面（分析中）

画面表示時に `GET /api/results/{user_id}` を呼び出し、結果データを取得する。

**必要な情報:**

- なし（API レスポンス待機中の演出のみ）

**表示パーツ:**

- 「分析中...」テキスト
- フェイクプログレスバー（0% → 95% まで自動進行）

**ローディング制御:**

- API レスポンス待機中はアニメーション継続
- 最低表示時間: 2 秒（レスポンスが早くても 2 秒は表示）

**エラー時:**

- 「結果の取得に失敗しました」等のエラーメッセージ
- 「もう一度取得」リトライボタン

**成功時:**

- 結果データを state（Jotai atom）に保存
- 自動的にフェーズ 2（行動解析REPORT）に遷移

**APIトリガー:**

| トリガー   | API                          | タイミング       |
| ---------- | ---------------------------- | ---------------- |
| 画面表示時 | `GET /api/results/{user_id}` | 画面を開いた瞬間 |

## フェーズ 2: 行動解析REPORT（タブ切り替え式）

Loading 完了後に表示。API から取得済みの `ResultResponse` を使用（追加 API 呼び出しなし）。

**必要なデータ（`ResultResponse` より）:**

| フィールド        | 型                            | 用途                                        |
| ----------------- | ----------------------------- | ------------------------------------------- |
| `scores`          | `DiagnosisScores`             | 実測 5 軸スコア（レーダーチャート）         |
| `baseline_scores` | `DiagnosisScores`             | 自己申告 5 軸スコア（レーダーチャート比較） |
| `gaps`            | `DiagnosisScores`             | 実測と自己申告の差分（ギャップ率算出）      |
| `feedback`        | `DiagnosisFeedback`           | タイトル・説明文・最大乖離項目              |
| `accuracy_score`  | `number`                      | 自己認識精度（0-100）                       |
| `phase_summaries` | `PhaseSummaries`              | 各ゲームの行動要約テキスト                  |
| `details`         | `ResultDetails`（game_1/2/3） | 各ゲームの特徴スコア・行動メトリクス        |

### ページ共通パーツ

- **ヘッダー:** 「行動解析REPORT」（カードの外側に配置）
- **タブナビゲーション:** 4 タブ切り替え（`総合診断` / `規約の罠` / `AIバトル` / `空気読み`）
- **ボタン群:**

| ボタン                     | 動作                                              |
| -------------------------- | ------------------------------------------------- |
| もう一度診断               | `localStorage.clear()` → トップ画面 `/` に遷移    |
| ゲーム結果の詳細を閲覧する | 「規約の罠」タブ（最初のゲーム詳細）に切り替え    |
| 結果をシェア               | シェアパネル表示（X / LINE / URLコピー / OS標準） |

---

### タブ 1: 総合診断

**必要な情報:**

- `baseline_scores` + `scores`（5 軸の比較用）
- `gaps`（ギャップ率算出用）
- `feedback.description`（フィードバック説明文）
- `feedback.gap_point`（最大乖離項目名）

**表示パーツ:**

| パーツ                 | 種類                | 内容                                                            |
| ---------------------- | ------------------- | --------------------------------------------------------------- |
| 五角形レーダーチャート | recharts RadarChart | 5 軸（慎重さ / 冷静さ / 論理性 / 協調性 / 積極性）の 2 系列表示 |
|                        |                     | 系列 1: 自己申告（`baseline_scores`、青系、破線）               |
|                        |                     | 系列 2: 実測値（`scores`、ピンク系、実線）                      |
|                        |                     | ホバー時ツールチップ: 軸名・自己申告値・実測値                  |
| ギャップ率表示         | テキスト            | 5 軸の `gaps` から最大絶対値を算出し、パーセント表示            |
| フィードバック文       | テキスト            | `feedback.description`（API から取得した説明文をそのまま表示）  |
| 最大乖離項目           | テキスト            | `feedback.gap_point`（最もギャップが大きかった項目名）          |

---

### タブ 2〜4: ゲーム詳細（規約の罠 / AIバトル / 空気読み）

3 タブとも同一コンポーネント。表示するデータが異なる。

**タブとデータの対応:**

| タブ     | データソース     | コメント                  | バーの色   |
| -------- | ---------------- | ------------------------- | ---------- |
| 規約の罠 | `details.game_1` | `phase_summaries.phase_1` | 赤系       |
| AIバトル | `details.game_2` | `phase_summaries.phase_2` | オレンジ系 |
| 空気読み | `details.game_3` | `phase_summaries.phase_3` | 青系       |

**必要な情報（各ゲームごと）:**

- `details.game_N.feature_scores[]`（特徴スコア配列: 軸名 + スコア 0-100）
- `details.game_N.metrics[]`（行動メトリクス配列: ラベル + ユーザー値 + 平均値）
- `phase_summaries.phase_N`（行動要約テキスト）

**表示パーツ（2 カラムレイアウト / モバイルは縦積み）:**

| パーツ               | 種類               | 内容                                                    | 配置 |
| -------------------- | ------------------ | ------------------------------------------------------- | ---- |
| 特徴スコアバー       | 水平プログレスバー | `feature_scores` の各項目を 0-100 のゲージで表示        | 左列 |
| 解析コメント         | テキスト           | `phase_summaries.phase_N`（そのゲームの行動要約）       | 左列 |
| 行動ログバーチャート | recharts BarChart  | `metrics` を横棒グラフで表示（あなた vs 平均の 2 系列） | 右列 |
|                      |                    | Y軸: メトリクスラベル、X軸: 数値                        |      |
|                      |                    | ホバー時ツールチップ: ラベル・ユーザー値・平均値        |      |
| 凡例                 | テキスト           | 「■ あなた」「■ 平均」                                  | 右列 |

## 5 軸の定義

| キー              | 日本語名 | レーダーチャート上の位置 |
| ----------------- | -------- | ------------------------ |
| `caution`         | 慎重さ   | 上                       |
| `calmness`        | 冷静さ   | 右上                     |
| `logic`           | 論理性   | 右下                     |
| `cooperativeness` | 協調性   | 左下                     |
| `positivity`      | 積極性   | 左上                     |

## シェア機能

「結果をシェア」ボタンで以下のシェア方法を選択可能:

| 方法             | 条件     | 動作                                   |
| ---------------- | -------- | -------------------------------------- |
| X (Twitter)      | 常時表示 | ツイート作成画面を新しいタブで開く     |
| LINE             | 常時表示 | LINE シェア画面を新しいタブで開く      |
| テキストをコピー | 常時表示 | シェアテキストをクリップボードにコピー |

**シェアテキスト:** `私の行動解析結果は「{feedback.title}」でした！ #行動解析REPORT`

> **NOTE:** OG 画像生成や URL 付きシェアは今後の追加機能として検討。現時点ではテキストのみ共有。

- `localStorage.getItem('user_id')` のコメントアウトを解除
